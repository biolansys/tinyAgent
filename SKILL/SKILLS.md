# General Skill

Use this repository's command model and project isolation rules.

- Work inside the active project under `workspace/<project>/`.
- Switch projects with `/project NAME` before project-specific work.
- Use `/edit FILE` for one-file changes.
- Use `/asksubagent worker` for scoped multi-file changes.
- Check `/gitstatus` before `/gitcommit`.
- Prefer `/snapshot` before larger changes.
- Keep edits small and reversible.
